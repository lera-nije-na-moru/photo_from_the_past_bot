# photo_from_the_past_bot

Инструмент для автоматического поиска **пропущенных постов рубрики "Фото из прошлого"** в Telegram-каналах.
Проект решает редакционную проблему:
разные администраторы публиковали рубрику по собственному алгоритму, из-за чего часть фотографий выпадала из ежегодного цикла.
Бот анализирует историю канала и находит посты, которые должны были выйти повторно (спустя год), но не вышли.

---

## Задача
В рубрике публикуются фотографии округа с подписью:

```
(всего X лет назад)
```
Условие повторной публикации:
- с момента последнего выхода должно пройти минимум 365 дней
- текст поста должен быть идентичен (с учётом нормализации)
  
Бот:
- собирает историю канала
- группирует посты по нормализованному тексту
- проверяет дату последней публикации
- отправляет список "пропущенных" постов администратору

---

## Архитектура

Используются два клиента:
- **Telethon user client** — для чтения истории канала
- **Telethon bot client** — для взаимодействия с админами

Это позволяет:
- обходить ограничения Bot API
- анализировать старые сообщения
- безопасно отправлять результаты в личные сообщения

---

## Логика работы

### Поиск постов рубрики

Регулярное выражение:
```python
pattern_years = re.compile(r"\(всего \d+ (лет|год|года) назад\)", re.IGNORECASE)
```
Фильтр:
- только посты с этой формулировкой
- только начиная с 2023 года (последнее изменение стилистики)

---

### Нормализация текста

Перед сравнением:
- приведение к нижнему регистру
- удаление ссылок
- удаление цифр
- удаление эмодзи и спецсимволов
- нормализация пробелов
---

### Группировка постов

```
normalized_text → [(date, id, original_text)]
```

Для каждого набора:

- сортировка по дате
- проверка, был ли пост менее года назад
- если нет — считается пропущенным

---

### Выдача результата

Пользователю отправляется список:

```
• 12.03.2023 — оригинальный текст

[Ссылка на пост]
```

## Интерфейс

После команды `/start` администратор выбирает канал. Бот запускает анализ и отображает прогресс каждые 500 сообщений (так как постов много, чтобы не было тревоги, что что-то сломалось).

## Ограничения

- Если текст поста редактировался (например, объект с фотографии был снесён или реконструирован), нормализация может не распознать его как тот же самый.
- Алгоритм ориентирован на текст, а не на фото-хэш.
- Возможны редкие false positives при совпадении формулировок.
- Необходима ручная проверка выдачи, но по опыту использования - заметно облегчает задачу и позволяет раз в месяц собирать запас фотографий в отдельный рабочий чат.
